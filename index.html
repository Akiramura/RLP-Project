How it works

From lobby to game
in seconds.

RLP runs silently in the background and activates exactly when you need it.

01
Open RLP alongside your League client
Launch the app once. It reads your local League client credentials automatically ‚Äî no login, no account creation required.
02
Queue up and enter champion select
The moment champion select begins, RLP detects it and switches to the Auto Import tab automatically. A pulsing indicator shows it's active.
03
Pick or hover a champion
As soon as your champion is detected, RLP fetches the optimal build data for your champion and assigned role from aggregated meta sources.
04
Runes, spells, and items are imported
RLP writes the rune page directly to your client via the LCU API, sets your summoner spells, and saves an item set. You're ready to play.

Privacy Policy

Last updated: February 2026 Current

1. Overview

RLP Project ("the App") is a free desktop application built with Tauri and React. This policy explains exactly what data the App accesses, how it is processed, and what is stored ‚Äî with no omissions.

2. Data We Do NOT Collect

The App does not collect your Riot account credentials or passwords, payment information, device identifiers, or any sensitive personal data. The App uses no cookies, advertising networks, analytics SDKs, or tracking of any kind.

3. Data Accessed Locally (Never Transmitted)

To detect champion select, the App reads the LCU lockfile that the League of Legends client writes to your local disk while running. This file contains a local port and authentication token used exclusively to make API calls to your own League client on 127.0.0.1. This data is processed entirely on your machine and is never transmitted to our servers or any third party.

4. Data Retrieved from Third-Party APIs

The App makes requests to the following external APIs on your behalf:

  • Riot Games API (official, authenticated with your API key) ‚Äî The App retrieves your Riot ID / PUUID, summoner profile, ranked entries, match history, champion masteries, and live game data. These requests are governed by the Riot Games Privacy Notice. The following endpoints are used: account/v1, lol/summoner/v4, lol/league/v4, lol/match/v5, lol/spectator/v5, lol/champion-mastery/v4.
  • OP.GG MCP API (mcp-api.op.gg) ‚Äî The App queries OP.GG's official public MCP API (MIT-licensed, published by OP.GG Inc. at github.com/opgginc/opgg-mcp) to retrieve champion builds, summoner spell recommendations, rune configurations, tier list data, and match history. The tools used are: lol_get_champion_analysis, lol_list_lane_meta_champions, lol_get_summoner_profile, lol_list_summoner_matches. No personal data beyond your Riot ID is transmitted in these requests.
  • Data Dragon CDN (ddragon.leagueoflegends.com) ‚Äî Champion and spell images are loaded directly from Riot's official CDN. No personal data is transmitted.

5. Server-Side Cache (Neon PostgreSQL)

To reduce redundant API calls and improve load times, the App uses a serverless PostgreSQL database hosted on Neon as a temporary cache. This cache stores only data that is already publicly accessible via the Riot Games API:

  • Summoner cache ‚Äî PUUID, Riot ID (game name + tag), summoner profile, ranked entries, and match history. This data is considered stale and refreshed after 60 minutes.
  • Match cache ‚Äî Individual match details (participants, items, KDA, game duration) indexed by match ID. Match results do not change, so this data is cached without a TTL to avoid redundant Riot API calls.

All cached data originates from the public Riot Games API. It is never sold, shared with third parties, or used for advertising or profiling. You may request deletion of your cached data by opening an issue on our GitHub repository with your Riot ID.

  • Riot Games ‚Äî Riot Games Privacy Notice
  • OP.GG ‚Äî OP.GG Privacy Policy
  • Neon ‚Äî Neon Privacy Policy

7. Children's Privacy

The App does not knowingly collect personal data from any user. The data cached server-side is sourced entirely from Riot's public API and is not linked to a real-world identity beyond the Riot ID.

8. Changes to This Policy

If this policy changes, the updated version will be published here with a new "Last updated" date. Continued use of the App after changes constitutes acceptance of the updated policy.

9. Contact
