How it works

From lobby to game
in seconds.

RLP runs silently in the background and activates exactly when you need it.

01
Open RLP alongside your League client
Launch the app once. It reads your local League client credentials automatically ‚Äî no login, no account creation required.
02
Queue up and enter champion select
The moment champion select begins, RLP detects it and switches to the Auto Import tab automatically. A pulsing indicator shows it's active.
03
Pick or hover a champion
As soon as your champion is detected, RLP fetches the optimal build data for your champion and assigned role from aggregated meta sources.
04
Runes, spells, and items are imported
RLP writes the rune page directly to your client via the LCU API, sets your summoner spells, and saves an item set. You're ready to play.

Privacy Policy

Last updated: February 2026 Current

1. Overview

RLP Project ("the App") is a free, open-source desktop application built with Tauri and React. We are committed to protecting your privacy. This policy explains what data the App accesses, how it is used, and what we do not collect.

2. Data We Do NOT Collect

We do not collect your Riot account credentials or passwords, payment information, IP addresses, device identifiers, or any sensitive personal data. We do not use cookies, advertising networks, or analytics SDKs of any kind.

3. Data Accessed and Cached

To function, the App retrieves publicly available League of Legends data via the Riot Games API and caches it to improve performance and enable offline access. Specifically:

  • League Client credentials ‚Äî RLP reads the local LCU (League Client Update) API lockfile while the League client is running on your machine. This data is used only to authenticate local API calls and never transmitted to our servers.
  • Riot Games API ‚Äî The App retrieves your summoner profile, ranked entries, match history, and champion masteries via the official Riot Games API. These requests are governed by Riot Games' Privacy Notice.
  • Third-party build data ‚Äî The App fetches anonymized champion build statistics from public meta sources (OP.GG). No personal data is sent in these requests.

4. Data Storage ‚Äî Neon Cloud Cache

To reduce redundant Riot API calls and enable offline access, the App uses a serverless PostgreSQL database hosted on Neon as a temporary cache.

Everything stored in this cache is entirely public data already freely accessible to anyone via the official Riot Games API ‚Äî the same data visible on sites like OP.GG, U.GG, and Blitz. This includes summoner profiles, ranked entries, match history, and match details. No private, sensitive, or non-public information is ever stored.

Summoner data is automatically considered stale and refreshed after 60 minutes. Match data is immutable and cached indefinitely since past match results never change. This cache exists purely for performance and is never used for advertising, profiling, or any purpose beyond serving the App.

You may request deletion of your cached data at any time by opening an issue on our GitHub repository with your summoner name.

5. Third-Party Services

The App makes requests to:

  • Riot Games API ‚Äî Riot Games Privacy Notice
  • Neon (neon.tech) ‚Äî serverless PostgreSQL cache ‚Äî Neon Privacy Policy
  • Data Dragon CDN (ddragon.leagueoflegends.com) ‚Äî champion images and game assets (no personal data transmitted)
  • OP.GG ‚Äî aggregated champion build statistics (no personal data transmitted)

6. Children's Privacy

The App does not knowingly collect any data from anyone, including minors. Because we collect no personal data at all, there is no special risk to children's privacy.

7. Changes to This Policy

If this policy changes, the updated version will be published here. Continued use of the App after changes constitutes acceptance of the new policy.

8. Contact

For privacy-related questions, open an issue on our GitHub repository.
